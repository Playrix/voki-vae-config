# voki-vae-config
Remote Config — механизм аварийного отключения модулей расширения без публикации новой версии. Это критически важно, потому что:

* Asana меняет DOM-структуру без предупреждения
* Контент-скрипты VAE могут крашить страницу для всех пользователей одновременно
* Нужна возможность мгновенно выключить неработающий модуль

Решение: JSON-конфиг на GitHub + кэш в chrome.storage.local с TTL 5 минут.

---

# Архитектура
```
GitHub (vae-config.json)
    ↓ (fetch на demand, если кэш > 5 min)
chrome.storage.local
    ↓ (chrome.runtime.sendMessage)
Service Worker (background.js)
    ↓
Content-scripts + Popup UI
```

---

# Поля

| Поле | Тип | Описание |
|---|---|---|
| `version` | string | Версия конфига (для future-compatibility) |
| `globalEnabled` | boolean | Глобальный рубильник (отключит все модули) |
| `modules` | object | Статус отдельных модулей |
| `modules[name].enabled` | boolean | `true` → модуль работает, `false` → отключён |
| `modules[name].reason` | string | Причина отключения (видна пользователю в popup) |
| `updatedAt` | ISO 8601 | Когда последний раз обновлялся конфиг |

---

# Примеры

**Отключить один модуль (DOM изменился):**
```json
"workloadHolidays": {
  "enabled": false,
  "reason": "Asana изменила структуру таймлайна, ожидание фикса"
}
```

**Отключить всё (критическая ошибка):**
```json
"globalEnabled": false
```

**Включить после фикса:**
```json
"workloadHolidays": {
  "enabled": true,
  "reason": ""
}
```

# Кэширование (TTL)

| Сценарий | TTL | Поведение |
|---|---|---|
| Кэш свежий (<5 мин) | — | Вернуть кэш, сеть не опрашивается |
| Кэш устарел (>5 мин) | 5 минут | Fetch из raw.git, обновить кэш |
| Сетевая ошибка | — | Вернуть старый кэш если есть, иначе `{ globalEnabled: true }` |
| Первый запуск | — | Fetch, сохранить в кэш |

**Когда обновляется кэш:**
- Пользователь открывает новую вкладку Asana
- Пользователь открывает popup
- Пользователь обновляет страницу Asana
- Прошло >5 минут с последнего fetch
